1. 概要 — このファイルは何か

541 個のエントリはすべて同じ構造で、「1 サーバー = 1 モデル（または 1 機能）」の対応になっている。

  • 接続方式：全件 type: http。エンドポイントは https://kamui-code.ai/… に集約されている（リモート MCP。ローカルにプロセスを立てない）
  • 認証：全件共通のヘッダー KAMUI-CODE-PASS に環境変数 ${KAMUI_CODE_PASS_KEY} を渡す方式。キー 1 本で全モデルが使える
  • 命名規則：入出力タイプ-kamui-モデル名。例：t2i（Text→Image）、i2v（Image→Video）、v2v（動画加工）、flf2v（最初と最後のフレーム→動画）、i2i3d（画像→3D）など。名前を見れば用途がわかる
  • ⭐マーク：説明文末尾の星（0〜5 個）は Kamui Code 側が付けた推奨度。モデル選定の実用的なガイドになる
  • 収録モデル：Veo 3.1、Sora 2/Pro、Kling 2.5/2.6/3.0/O1、Seedance 1.x/2.0、Hailuo 2.3、Flux-2（Pro/Max/LoRA）、Qwen Image 2、GPT Image 1.5、Hunyuan3D 3.1、ElevenLabs、MiniMax Speech 2.6、Lyria 3 Pro など、2026 年時点の主要モデルをほぼ網羅

4. PROS / CONS

PROS 強み

  • 圧倒的な網羅性と鮮度Sora 2、Veo 3.1、Kling 3.0/O1、Seedance 2.0、Flux-2、Hunyuan3D 3.1 など、リリース直後の最新モデルが 541 件分そろう。個別に各社と契約する必要がない
  • 統一インターフェース全サーバーが同じ HTTP MCP + 共通ヘッダー認証。キー 1 本・書式 1 種類で画像・動画・音声・3D・LoRA 学習まで扱える
  • 命名規則が自己文書化されているt2i / i2v / flf2v のプレフィックスで入出力が即座にわかり、モデル比較・乗り換えが容易。ベンダーロックインが弱い
  • 星評価つきカタログ541 件から選ぶ際のガイドが組み込まれている。★4 以上の 70 件に絞れば選定コストが大きく下がる
  • 制作パイプラインが一気通貫LoRA 学習 → 画像生成 → 画像編集 → 動画化 → リップシンク → 字幕 → アップスケールまで、1 つのエコシステム内で完結できる
  • 詳細なパラメータ記載（一部）新しいエントリは解像度・秒数・上限サイズ・デフォルト値まで description に明記されており、ドキュメントを見ずに呼べる

CONS 弱み・注意点

  • 全部入り登録はコンテキストを破壊する541 サーバーをそのまま MCP 登録すると、ツール定義だけで LLM のコンテキストを大量消費し実用にならない。用途ごとに数個だけ選んで登録する運用が前提（本 workspace では /mcp-select で選定）
  • 単一ゲートウェイ依存全 541 件が kamui-code.ai 経由。ここが落ちる・仕様変更する・サービス終了すると全機能が同時に止まる
  • キー 1 本の集中リスク1 つのキーで全モデルが呼べる = 漏洩時の影響範囲が最大。環境変数管理の徹底と、キーを書いた設定ファイルの公開リポジトリ混入防止が必須
  • コストが見えにくい裏側のモデルごとに課金が異なるのに、MCP 越しには単価が見えない。動画系（特に Pro / Max / 1080p）は高額になりやすく、うっかり連打が怖い
  • 説明の粒度がバラバラ詳細なエントリと「via Kamui Code ⭐」だけの 1 行エントリが混在。★1 の約 80 件は情報不足のまま選ぶことになる
  • 新旧バージョンの乱立veo3 / veo31、seedance v1 / v1.5 / v2.0 など類似名が併存し、古い方を誤って選びやすい。星評価と世代名の確認が必要
  • 生成時間・キュー待ちはモデル依存3D（Meshy は 5〜10 分）や高品質動画は待ち時間が長く、同期的なワークフローに組み込みにくい

5. 運用の指針（推奨）

  • 登録は「タスクごとに数個」：この JSON はカタログとして保管し、実際の .mcp.json にはその日のタスクに必要なサーバーだけ入れる。選定は ★4〜5 から始める
  • デフォルトの推し：画像→動画は i2v-kamui-veo31（品質）か veo31-fast（速度）、画像生成は flux-2-max か gpt-image-1-mini、画像編集は qwen-image-2-edit か flux-2-max-edit、TTS は minimax-speech-26-hd
  • キー管理：KAMUI_CODE_PASS_KEY は環境変数のみ。この JSON 自体は変数参照なのでそのまま共有可能だが、展開済みの値を含むファイルは絶対に公開しない
  • コスト対策：高額な動画系はまず Fast / Lite / 480p 版でプロンプトを固めてから、本番だけ Pro / 1080p を叩く二段構えにする
ひとことで言うと：Kamui Code は「生成 AI モデルの総合商社」。品揃え・鮮度・使い勝手の統一感は非常に高い。一方で「全部つなぐ」ものではなく「選んでつなぐ」ものであり、コンテキスト管理・キー管理・コスト管理の 3 点を自分で握る前提のサービス。
