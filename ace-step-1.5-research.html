エグゼクティブサマリー

ACE-Step 1.5（2026年1月リリース）は、MITライセンスで完全無料・完全ローカル動作するオープンソース音楽生成AIモデル。音楽AIの「Stable Diffusion moment」と称され、これまでクラウドAPIに閉じていた商用品質の音楽生成を、コンシューマーGPUへ開放した。


2026年4月にリリースされた上位版 ACE-Step 1.5 XL（4Bパラメータ）は、11項目の主要ベンチマークすべてでSuno v5を上回るスコアを記録。RTX 3090で10秒以下・A100で2秒以下という驚異的な速度を実現している。

4.2 強化学習（内在的報酬）

外部の人間フィードバックモデルを使わない内在的強化学習を採用。クロスアテンションヘッド間の一致度を報酬信号として利用し、歌詞とオーディオの同期を自己最適化する。

インストール方法

8.1 標準インストール（Gradio UI）

# Step 1: uvのインストール
curl -LsSf https://astral.sh/uv/install.sh | sh

# Step 2: リポジトリのクローン
git clone https://github.com/ace-step/ACE-Step-1.5
cd ACE-Step-1.5

# Step 3: 依存関係のインストール
uv sync

# Step 4: 起動（初回はモデルを自動ダウンロード）
uv run acestep

8.2 モデルのダウンロード

# 主要モデルのダウンロード
uv run acestep-download

# 全モデルをダウンロード（時間がかかる）
uv run acestep-download --all

# ModelScopeから（中国・特定地域向け）
uv run acestep-download --download-source modelscope

8.3 ComfyUI 経由（推奨：ビジュアルワークフロー）

ダウンロードファイルace_step_1.5_turbo_aio.safetensors（All-in-One版）
配置先ComfyUI/models/checkpoints/
ワークフロー取得ComfyUI のテンプレートから「ACE-Step」を検索

限界・既知の問題

音質面

問題詳細重症度
ボーカルアーティファクト メタリックな音色・サ行の歪み・ピッチのズレ（特に高域・ファルセット） 中〜高
クリッピング/歪み ロック・エレクトロ系で顕著。ステップ数50→100で改善するが生成時間が倍 中
表現力不足 ビブラート・ピッチベンドなどの連続的なニュアンスが乏しい 中
メロディの記憶残存性 技術的には正確だが印象に残りにくいメロディ 低〜中
ドラムの機械的感 感情的なドラムフィルが弱い。エントリーが唐突 低〜中

機能面

問題詳細重症度
Repaintモード 差し替え区間にボーカルが入らないことが多い。前後との不自然なつなぎ目（Issue #1151） 中〜高
歌詞追従性 英語歌詞の追従精度が低いと報告（Issue #391） 中
ジャンルドリフト 2分超の生成でジャンルが変わることがある 中
シード依存性 ランダムシードで品質が大きく変動する「ガチャ」状態 中
XL版のVRAM要件 2B版は4GBで動作するが、XL（4B）は12〜20GB必要 低（上位GPU所有者向け）

コミュニティ動向

KVR Audio フォーラム

Text2Music・カバー編集・楽器抽出などの多機能性を高く評価する声が多い。RTX 3070 8GB でも動作するという報告あり。一方、「ボタン押して曲が出る」アプローチへの懐疑的なコメントも散見。複数バージョン（GitHub無料版・オンライン有料版・ACE Studio）の混乱を指摘するユーザーも。

GitHub コミュニティ

Issue 1,000件超と非常に活発。コミュニティから ComfyUI ノード・ONNX エクスポート・REST APIラッパーが提供されている。Musician's Guide（Discussion #235）など、使用者向けの詳細なディスカッションが充実。

プロデューサー層の動き

一部プロデューサーが「サブスクを切ってローカル移行」する動きを報告。LoRA によるスタイル個人化が最大の差別化要因として認識されている。AMD公式がサポートを発表したことで、NVIDIA 以外のユーザー層への普及も加速している。

プロデューサーへの推奨

✓ ACE-Step 1.5 が向いているケース

  • ローカル環境でプライバシーを守りながら音楽生成したい
  • MITライセンスで商用利用したい
  • 独自スタイルをLoRA学習させてオリジナリティを出したい
  • ComfyUI でノードベースのワークフローを組みたい
  • 長尺（最長10分）の楽曲を生成したい
  • コスト0でサブスク不要にしたい
  • AMD / Intel / Apple Silicon を使っている

✗ ACE-Step 1.5 が向いていないケース

  • ボーカルクオリティを最優先する（Suno v5が優勢）
  • セットアップなしですぐ使いたい
  • 高スペックなGPUを持っていない（XL版は12〜20GB必要）
  • 歌詞追従精度が命（現状は課題あり）
  • Repaint機能を多用する予定（現状の品質に課題あり）
💡 結論: ローカル・無料・カスタマイズ性で圧倒的に優れており、音楽制作の「Stable Diffusion的存在」になりつつある。ボーカル品質の課題が今後のアップデートで改善されれば、商用ツールの完全代替が視野に入る。
